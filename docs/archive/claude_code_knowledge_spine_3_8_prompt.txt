Mission:
Upgrade Karma into a self-feeding local-first knowledge system that can:
1. navigate structured sites
2. harvest documentation
3. ingest raw dropped files with minimal user prep
4. support saved-page digestion including MHT/MHTML
5. import repo explainer notes
6. build a reusable local knowledge library
7. learn from its own code and patches
8. prepare for semantic retrieval
9. add code intelligence so it can reason about codebases instead of only reading files

PHASE 1 — Navigator Spine (Wikipedia-first)
Stabilize and extend the navigator subsystem.

Required capabilities:
- open URL
- extract page content
- detect and score internal links
- follow relevant links only
- avoid loops
- obey max depth / max pages / same-site rules
- record stop_reason

Support commands/behaviors such as:
- navigate wikipedia <topic>
- navigate site <url>
- golearn wikipedia <topic>

Wikipedia rules:
- prefer article body and section text
- ignore edit/history/meta/admin/help/login pages
- follow useful /wiki/ article links only
- stop when yield drops

PHASE 2 — Documentation Harvester Spine
Add a documentation harvester for structured technical docs.

Target sites first:
- docs.python.org
- kali.org/docs
- debian.org/doc
- stable official project docs
- readthedocs-style sites when practical

Required capabilities:
- harvest from a seed URL
- recognize docs pages
- preserve headings
- preserve code blocks and command examples
- follow documentation links only
- stop when enough useful docs are collected
- save cleaned pages locally with manifest/provenance

Support commands/behaviors such as:
- harvest docs <url>
- harvest python docs
- harvest kali docs
- harvest debian docs

Use these project-level features to keep token use down and make sessions more reliable:
- Put project instructions in ./CLAUDE.md. Anthropic documents project memory in CLAUDE.md and recommends /init to bootstrap it.
- Put reusable prompts in .claude/commands/ as custom slash commands.
- Use .claude/settings.json and .claude/settings.local.json for project/shared vs local settings, including permission rules and extra directories.
- Use focused subagents with limited tool access for narrow jobs instead of one giant do-everything agent.
- Use hooks for deterministic actions like post-edit formatting, validation, or blocking unsafe writes.
- Use /compact to shrink long sessions while preserving focus, instead of letting context bloat.
- Keep prompts narrow and stage-gated. Require runtime proof, not just passing tests.
