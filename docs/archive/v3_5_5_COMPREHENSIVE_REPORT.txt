3. IMPLEMENTATION DETAILS
================================================================================

RESEARCH/INGESTOR.PY:
--------------------

SUPPORTED FILE EXTENSIONS:
  .md, .txt, .json, .py, .rst, .html, .csv
  .yaml, .yml, .toml, .sh, .js, .ts, .c, .cpp
  .h, .java, .go, .rs, .sql, .xml, .mdx

KEY CLASSES:

@dataclass
class IngestedItem:
    id: str                         # Unique ID (ing_00001)
    source_path: str               # Original file path
    topic_bucket: str              # Classified topic
    provenance: str               # seed_pack, local_doc, etc.
    file_type: str                # .md, .py, etc.
    content_hash: str              # SHA256 prefix for dedup
    imported_ts: str                # ISO timestamp
    text_length: int               # Content length
    title: str                     # Extracted title
    content: str                   # Full text content
    metadata: Dict                 # Extra metadata

@dataclass
class IngestStats:
    files_scanned: int
    files_accepted: int
    files_rejected: int
    duplicates_skipped: int
    topic_counts: Dict[str, int]
    provenance_counts: Dict[str, int]
    errors: List[str]

KEY METHODS:

SeedIngestor.ingest_path(source_path, move_processed, move_rejected)
  - Scans folder recursively
  - Filters by extension
  - Extracts text content
  - Classifies topics
  - Detects provenance
  - Deduplicates by hash
  - Saves to manifest

SeedIngestor.search_by_topic(topic: str) -> List[IngestedItem]
  - Returns all items matching topic

SeedIngestor.search_local(query: str, topics: Optional[List]) -> List[IngestedItem]
  - Full-text search across local knowledge

GRMAR PATTERNS:
--------------
- ingest <path>
- import <path>
- load <path>
- seed <path>
- golearn ingest <path>
- karma ingest <path>

AGENT HANDLER:
-------------
_run_ingest(params) -> Dict
  - Gets path from params
  - Calls ingestor.ingest_path()
  - Formats stats as markdown output
  - Returns success/error

5. USAGE GUIDE
================================================================================

STEP 1: CREATE KNOWLEDGE FOLDER
-------------------------------

Create a folder with this structure:

knowledge_drive/
  00_inbox/           <- General notes
  01_kali_linux/      <- Kali Linux docs
  02_python/         <- Python docs
  03_debugging/       <- Debugging guides
  04_ai_frameworks/   <- AI/ML docs
  05_coding_patterns/  <- Patterns/architecture
  06_systems/         <- Systems topics
  07_docs_reference/  <- Manuals/reference
  08_seed_packs/      <- Curated seed packs
  09_processed/      <- Auto-moved after ingest
  10_rejected/        <- Auto-moved if rejected

STEP 2: ADD FILES
-----------------

Drop your files into the appropriate folders. Supported types:
- Markdown (.md)
- Text (.txt)
- Code (.py, .js, .go, .rs, etc.)
- JSON (.json)
- HTML (.html)
- Others (see SUPPORTED_EXTENSIONS)

STEP 3: INGEST
--------------

Run any of these commands:

  ingest /path/to/knowledge_drive
  import /path/to/folder
  load /path/to/folder
  seed /path/to/folder
  karma ingest /path/to/folder
  golearn ingest /path/to/folder

STEP 4: VIEW RESULTS
--------------------

Output shows:
- Files scanned/accepted/rejected
- Topic breakdown
- Provenance breakdown
- Total items in knowledge base

STEP 5: SEARCH
-------------

Use in code:
  from research.ingestor import get_ingestor
  ingestor = get_ingestor()
  results = ingestor.search_local("python decorators")
  items = ingestor.search_by_topic("python")

7. TOPIC CLASSIFICATION
================================================================================

TOPIC BUCKETS:

kali_linux     <- From: kali, kali_linux, penetration, security
python         <- From: python, py_
debugging      <- From: debugging, debug, troubleshoot
ai_frameworks  <- From: ai, ml, llm, torch, tensorflow
coding_patterns <- From: patterns, architecture, design, algorithms
systems        <- From: systems, os, linux, network
docs_reference <- From: docs, reference, manual, tutorial, guide
misc           <- Default fallback

CLASSIFICATION METHOD:

1. PATH-BASED (primary):
   - Check folder names for topic keywords
   - "01_python" -> python
   - "03_debugging" -> debugging

2. CONTENT-BASED (secondary):
   - Scan first 5000 chars of content
   - Look for topic keywords
   - "def " + "async" -> python
   - "nmap" or "metasploit" -> kali_linux
