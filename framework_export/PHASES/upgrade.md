# Upgrade

- Phase purpose: add capability with controlled creativity and operational discipline.
- Preferred active roles: `Builder`, `Validator`.
- Preferred support roles: `Dreamer`, `Forecaster`, `Scope Guard`.
- Acceptable change size: moderate if planned and validated; small if confidence is weak.
- Common risks: innovation drift, adoption friction, vague value claims, phase mismatch.
- Escalation rules: escalate when upgrades threaten delivery, fail validation, or trigger repeated churn; enter triage if regressions appear.
- Success shape: meaningful capability gain, clear adoption path, and no uncontrolled bloat.
