# Prototype

- Phase purpose: prove value quickly with bounded risk.
- Preferred active roles: `Builder`, `Scout`.
- Preferred support roles: `Validator`, `Scope Guard`.
- Acceptable change size: moderate if isolated and reversible.
- Common risks: overbuilding, weak validation, early architecture lock-in.
- Escalation rules: escalate when the prototype starts spreading into core paths or confidence collapses.
- Success shape: usable proof with clear limits, not a polished system.
