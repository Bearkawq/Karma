# Playbooks Index

| Name | Use | Who Uses It | Status |
| --- | --- | --- | --- |
| Repeated Failure Playbook | when the same problem keeps returning | Checker, Helper | active |
| Stuck Task Playbook | when work stalls or loses direction | Helper, Planner | active |
| Handoff Recovery Playbook | when a handoff is thin or low confidence | Scout, Checker | active |

## Repeated Failure Playbook
- stop adding new changes
- compare the last attempts
- isolate what stayed broken
- ask Checker for the exact failure point
- reassign only after the evidence is clear

## Stuck Task Playbook
- restate the task, blocker, and current loadout
- shrink scope to one next move
- activate Helper if the owner is looping
- use a recovery item before opening new work

## Handoff Recovery Playbook
- read the handoff only once
- pull the listed files and Armory item
- separate known facts from guesses
- produce a tighter next step before acting
