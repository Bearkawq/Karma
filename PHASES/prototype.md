# PHASES/prototype.md

## Phase: Prototype

### Purpose
Explore freely, try approaches, prove concepts work.

### Preferred Active Roles
- Builder (primary)
- Scout (support)

### Preferred Support Roles
- Forecaster (predict next)
- Validator (early idea screening)

### Acceptable Change Size
**High** — Explore, experiment, iterate.

### Common Risks
- Scope creep
- Forgetting to test properly
- Over-engineering before validation

### Escalation Rules
- If concept proven, transition to patch
- If stuck, call Stabilizer
- If idea emerges, activate Dreamer -> Validator

### Success Shape
- Working proof of concept
- Tests pass (or known failures documented)
- Clear next steps identified

*Use when building something new or exploring approaches.*
