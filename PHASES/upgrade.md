# PHASES/upgrade.md

## Phase: Upgrade

### Purpose
Add significant new capabilities, version upgrades, major improvements.

### Preferred Active Roles
- Builder (primary)
- Dreamer (support)

### Preferred Support Roles
- Validator (idea screening)
- Scope Guard (prevent overbuild)

### Acceptable Change Size
**High** — Significant but validated.

### Common Risks
- Breaking compatibility
- Scope explosion
- Over-engineering
- Introducing regressions

### Escalation Rules
- If Dreamer has ideas, activate Validator before implementing
- If breaking change detected, pause and consult Planner
- If test failures, transition to patch to fix

### Success Shape
- New capability working
- Tests pass
- Breaking changes documented

---
*Use when adding major features, upgrading versions, or significant system changes.*
