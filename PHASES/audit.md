# PHASES/audit.md

## Phase: Audit

### Purpose
Observe, analyze, document — no changes unless critical.

### Preferred Active Roles
- Faultfinder (primary)
- Scope Guard (support)

### Preferred Support Roles
- Scout (terrain mapping)
- Forecaster (predict implications)

### Acceptable Change Size
**None** — Observe and document only, unless critical regression found.

### Common Risks
- Analysis paralysis
- Missing critical issues
- Confusing observation with action

### Escalation Rules
- If critical bug found, transition to patch or surgery
- If improvement opportunity found, note in IDEALS.md
- If scope expands, Planner decides next phase

### Success Shape
- Clear findings document
- No new issues introduced
- Recommendations for next phase

---
*Use when investigating, reviewing, or assessing.*
